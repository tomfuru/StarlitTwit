StarlitTwit readme.txt 最終更新:2011/12/31

*概要
自分の好きな機能が入ったTwitterClientを作ろうと思い個人的に作り始めたクライアント．
ひと通りのことはできるはず．
バグに気づいたらメモして気が向いたら直しています．

*必要環境
.NET Framework4.0が必要です．
「アプリケーションの初期化に失敗しました」等のエラーが起動して出る場合は.NET Framework4.0がインストールされているか確認してください．

.NET Framework 4.0 Webインストーラー
http://www.microsoft.com/downloads/ja-jp/details.aspx?FamilyID=9cfb2d51-5ff4-4491-b0e5-b386f32c0992

*インストール/アンインストール
必要ありません．設定ファイル等も全てexeのあるフォルダの下に置かれます．不要になったらフォルダごと削除してください．

*始めに
初めて起動した時はメニューの「機能」→「認証」からユーザー認証を行ってください．

*機能
・Twitterのタイムラインの表示・投稿
 - 投稿ではURL短縮も可能(現在bit.ly/j.mpのみ対応)
   -> Twitterの仕様が変わり，クライアントでURL短縮しなくても内部で勝手にt.coで短縮されます(その場合表示する時には自動的に展開されて表示します)．
 - ホーム/リプライ/発言履歴/ダイレクトメッセージ
  * UserStream利用可
 - 自分or他人のリスト
 - 検索
 - 発言は「通常発言(デフォルト背景色:白)」，「自分へのリプライ(デフォルト背景色:赤)」，「他人へのリプライ(デフォルト背景色:青)」，「リツイート(デフォルト背景色:緑)」で表示されます(背景色・フォント設定可能)．
・タブで色々なタイムラインを管理
 - Home,Reply,History,Directは移動不可能だが他のタブはドラッグ&ドロップで移動できます

・リスト管理機能
 - リストの追加・編集・削除が可能(メニュー：プロフィール→自分のリスト)
 - ユーザーのプロフィール画面から自分のリストの追加が可能(プロフィール画面のメニュー：操作→リストに追加・削除する)
 - ユーザーのリスト一覧画面からリストのフォロー(subscribe)と解除(unsbscribe)が可能(リスト一覧画面の右クリックメニュー)

・発言の一部選択等がテキスト部分ダブルクリックで可能
 - URL,ハッシュタグ,@ユーザー名等があればそこからURL表示/ハッシュタグのタブ・ユーザーのタブ追加が可能
・ReplyのToolTip表示
 - デフォルトで青色の発言(誰かの発言へのリプライ)の発言部分にマウスを置くとその元発言を取得していれば表示される
  * もし(取得していないものも含む)会話全てを見たいのなら右クリックメニューの「会話を表示」で可能．しかし鍵付ユーザーの発言を含んでいると全て見れない可能性があります．
 - 発言をクリックすると表示されません，注意．

・画像サムネイル表示
 - 発言をクリックして，そこに画像へのURLがあれば，ロードしてサムネイル表示する
 - 現在twitpic,photozou,yFrog,img.ly,movapic,plixi,lockerz,ow.ly,ついっぷるフォト,instagram,youtube,ニコニコ動画に対応
 - 複数画像がある場合は一定間隔(デフォルト3秒)ごとに切り替わります
 - もしサムネイルが残ってしまったらそのサムネイルをクリックすると消えます．

・その他
 - プロフィールの変更（メニュー：プロフィール→自分のプロフィール）
 - ユーザー検索（メニュー：機能→ユーザー検索）
 - 指定時刻発言取得（メニュー：機能→時刻を指定して発言取得
 - 診断メーカーがブラウザを開かなくても診断可能(診断メーカーのURLをクリックすると専用画面が表示)

*仕様
・ヘッダ/フッタは引用/リプライ/DMにはつきません
・同じ設定ファイルで二重起動はできません
・現状プロキシをスルーするようにしているので問題が発生するかもしれません

*設定
・設定ファイル(Settings.dat)はただのxmlファイルなのでテキストエディタでいじることもできますが変にいじると読めなくなりますので注意．
・取得設定
 - タイムライン・リプライ・履歴・ダイレクトメッセージのタブの取得数・取得間隔を設定可能
 - 自分のプロフィールの取得間隔を設定可能
・表示設定
 - ツイートの上段表示フォーマットを設定可能(後述)
 - 引用の種類を設定可能
 - アイコンの表示有無，サイズを設定可能
・フォント・色設定
 - 通常発言，自分へのReply，他人へのReply，リツイートそれぞれについて背景色とフォントを設定可能
・サムネイル設定
 - 画像サムネイルの表示有無が設定可能
 - 画像が複数ある時の切り替え間隔を設定可能
 - 各サイトにおけるサムネイルの大きさ/表示有無を設定可能
・UserStream設定
 - 起動時にUserStreamを開始するか設定可能
 - UserStreamでAllReplies(フォロワー→他人,他人→フォロワーのリプライも取得する設定)を有効にするか設定可能
 - イベントがきた時にタスクトレイバルーンに表示するか設定可能
 - UserStreamの開始時ログ表示設定
・フィルタ設定
 後述
・その他設定
 - ヘッダ・フッタを設定可能
 - 確認ダイアログの有無を設定可能
 - Replyのツールチップの深さ設定
 - リプライ・DMのバルーン表示を設定可能
 - ウェブブラウザの設定
  * デフォルトに設定されたウェブブラウザ以外を使用したい時は，使用ウェブブラウザに実行ファイルのパスを設定

* 見つかっているバグ/修正予定のこと
・画像のサムネイル表示がうまくいかないことがある
・表示位置がおかしい画面がある
・フィルターを適用しても現在あるものに適用されない
・WindowsのVisualStyleをいじるとタブの表示がおかしくなって落ちることがある
・その他いっぱい

*上段表示フォーマット
発言者の名前・時刻などが出る部分を自分の好きなように設定できます．以下の特殊文字列を使うとデータに応じて置き換えられます．
$Locked$ : ロックされている時に'◆'(RT,DM,Searchは使用不可)
$Favorited$ : お気に入りの時に'★'(DM,Searchは使用不可)
$Name$ : ユーザー名
$ScreenName$ : アカウント名(Searchは使用不可)
$Source$ : 投稿クライアント名(DMは使用不可)
$Retweeter$ : リツイートした人のScreen名(RTのみ使用可)
$Recipient$ : DMを受け取った人のScreen名(DMのみ使用可)
$DateTime(yyyy/MM/dd hh:mm:ss)$ : 日付と時刻(括弧内はフォーマット,後述)
$RTDateTime(yyyy/MM/dd hh:mm:ss)$ : RT元発言の日付と時刻(括弧内はフォーマット,後述)(RTのみ使用可)
\n : 改行

・時刻の括弧内フォーマットについて
y:yが連続する部分に西暦の年が入る(多すぎる時は左が0に)
M:Mが連続する部分に月が入る(多すぎる時は左が0に)
d:dが連続する部分に日が入る(多すぎる時は左が0に)
tt:午前/午後
H:Hが連続する部分に時(0〜23)が入る(多すぎる時は左が0に)
h:hが連続する部分に時(1〜12)が入る(多すぎる時は左が0に)
m:mが連続する部分に分が入る(多すぎる時は左が0に)
s:sが連続する部分に秒が入る(多すぎる時は左が0に)

'm'という文字を出したい時は'\m'と打てばそのまま出ます．

(例："yyyy/MM/dd HH:mm:ss" の時 "2000/01/01 13:01:01"(2000年1月1日13時1分1秒)となる)

*フィルタ設定
・「有効」チェックを入れたもののみフィルタとして有効になります
・正規表現について
正規表現のテキストボックスをクリックすると正規表現入力フォームが出ます．
上段に正規表現を入力します．正しい正規表現の時のみOKが押せます．
確認用テキストは入力すると上段の正規表現でフィルタリングされるかどうか確認できます(合致でフィルタリングされ表示されなくなる)．

正規表現が分からなくても，基本的にそのままフィルタリングしたい単語を書けば大抵問題ないです．
（例：クライアントtwittbot.netからの発言をフィルタリングしたい時は，正規表現として"twittbot.net"とすればOK)
